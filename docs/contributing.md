# Contributing

Thanks for considering a contribution. The project is small and the priorities
are concrete — please skim this page before opening a PR.

## What's most useful right now

1. **Example courses.** `examples/` is empty by design (see
   [examples/README.md](../examples/README.md)). A genuine example must come
   from running the skill on a real course, not from hand-assembled artifacts.
   If you've used `course-maker` for teaching, a stripped-of-PII export is the
   most valuable contribution.

2. **Beamer error patterns.** `references/step4_slides.md` has an anti-overfull
   checklist but no catalog of specific compilation errors. If you hit a
   Beamer error and fix it, open a PR adding both the error signature and
   the fix to that file.

3. **Reference-file refinements.** Each `references/step*.md` and
   `references/lab_*.md` describes a workflow. If you ran a step and hit a
   case the reference didn't anticipate, propose an addition.

## What's out of scope right now

- Large architectural reworks before the active waves in
  [IMPROVEMENT_PLAN.md](IMPROVEMENT_PLAN.md) are done. Wave 4 (profiles
  layer) is the gate for proper multi-instructor / multi-LMS support.
- New languages in `skill/templates/` (currently `en` and `ru`). Adding
  `de` / `fr` / `es` will be easier after wave 7; before that, partial
  language support tends to break the lab pipeline silently.

## Skill conventions

- All text under `skill/SKILL.md` and `skill/references/` must be English.
  The skill is language-agnostic; per-course language lives in
  `course_conventions.md` and `lab_templates.md` (generated by the init
  wizards). See [CLAUDE.md](../CLAUDE.md) for the rationale.
- `SKILL.md` should stay ≤350 lines (raised from 300 once a 4th pipeline plus
  inline CRITICAL safety blocks made 300 unreachable without cutting safety
  rails). Full workflows live in `references/`; `SKILL.md` is a dispatcher plus
  the `## Inviolable rules` block. See
  the discussion of the sticky-rule pattern in
  [IMPROVEMENT_PLAN.md § Part II](IMPROVEMENT_PLAN.md).
- New reference files: add a thin dispatcher in `SKILL.md` that names the
  reference; add any critical-rule short list as a `**CRITICAL — even if
  reference was skipped:**` block if the step has a history of silent skips.

## PR checklist

- [ ] No Russian or other non-Latin text in `skill/SKILL.md` or
      `skill/references/` (use `scripts/nonlatin.py skill/` to check)
- [ ] `wc -l skill/SKILL.md` < 350
- [ ] If you added a command: it appears in both `SKILL.md` Quick Reference
      tables and in `README.md` Commands tables
- [ ] If you added a reference file: `SKILL.md` has a dispatcher for it
- [ ] If you changed a workflow: corresponding `history.md` examples in
      `references/` still reflect the new flow

## Reporting bugs

Open an issue on GitHub. For Beamer/figure issues include the full error
output and the offending `.tex` or `.py` snippet. For pipeline issues
include the failing command and the relevant `history.md` entries.
