# Template Migration Plan — Language-Agnostic Skill

**Goal:** Move all language-specific and professor-specific content out of skill
reference files into per-course template files. The skill itself becomes fully
language-agnostic. Works with any LLM/framework — templates are plain files in
the course root, readable by any tool.

---

## New files introduced

### In `skill/templates/` (sources; copied by init wizard)

| File | Content |
|---|---|
| `course_conventions_ru.md` | Russian language rules + terminology dict |
| `course_conventions_en.md` | English equivalent (minimal dict) |
| `lab_templates_ru.md` | Russian notebook header, Block 0, self-check cell, docstring format, TODO style, hint format, scoring strings |
| `lab_templates_en.md` | English equivalents |

### In course root (generated by wizard; one per course)

| File | Generated by | Content |
|---|---|---|
| `course_conventions.md` | `/course-maker course init` | Language rules + terminology for this specific course |
| `lab_templates.md` | `/course-maker lab course-init` | All notebook/test templates for this course |

The wizard detects the course language from the "Language of slides and speaker notes"
answer (already collected during `course init`) and copies the matching template.
No new questions for the professor.

## What moves where

### Content moving from `lab_context.md` → `lab_templates.md`

- **Notebook Structure** section (entire Block 0 template):
  - First markdown cell (header: lab title, Курс/Course, Цель/Goal, Тема/Topic, Дедлайн/Deadline)
  - Block 0 divider cell
  - Task 0.1 (Student_ID input cell with comment)
  - Task 0.2 (variant assignment code)
  - Task 0.3 (dependency install cells)
  - Second-to-last markdown cell (final checklist)
  - Last code cell (self-check: required_vars loop, bonus_vars loop, all print strings)
- **Task formatting** details:
  - Docstring format (language-specific fields: Параметры/Parameters, Возвращает/Returns)
  - TODO comment style (`# TODO: ваш код` / `# TODO: your code`)
  - Variable stub comment style (`# Часть А:` / `# Part A:`)
- **Scoring block marker** in conftest.py
  (`# СИСТЕМА ПОДСЧЁТА БАЛЛОВ ДЛЯ ЛАБОРАТОРНОЙ РАБОТЫ` / `# SCORING SYSTEM`)
- **Grade output string** in conftest.py
  (`ПРЕДВАРИТЕЛЬНАЯ ОЦЕНКА В ЖУРНАЛ` / `PRELIMINARY GRADE FOR JOURNAL`)
- **Hint format**: `> 💡 **Подсказка:** ...` / `> 💡 **Hint:** ...`
- **Bonus task marker**: `**Бонус:**` / `**Bonus:**`
- **Task title format**: `### Задание N.M — Название` / `### Task N.M — Title`
- **Section title for common problems** in datasets_info.md
  (`Частые проблемы и их решения` / `Common Issues and Solutions`)

### Content moving from `lab_context.md` → `course_conventions.md`

- **Language and Terminology** section:
  - Which language is used, when English is acceptable
  - Dual-abbreviation convention (HMM / СММ etc.)
  - First-mention rule
- **Terminology dictionary** (English ↔ course language)
- **"Never use"** terminology rules
  (e.g. "форма" for "размерность", "fit" without context, etc.)
- **Lab goal writing rule** bad/good examples (currently in Russian)

### Content staying in `lab_context.md` (language-agnostic)

- Role and Goal section
- Factual Claims rule (no unverified numbers)
- Points structure (Block 0 = 0 points, etc.)
- Library Currency rule (check PyPI, flag stale libraries)
- Type annotations requirement
- Visualization return type requirement (`matplotlib.figure.Figure`)
- Variant formula invariant (`dataset_id = (Student_ID - 1) % len(datasets)`)

All Russian text in this file falls into two types:

**Type 1 — structure labels** (section headers, table headers):

- `**Общее время:**` → `**Total time:**`

- `## Таблица тайминга` → `## Timing table` + annotation "(translate to course language)"
- `## Что можно сократить` → `## What can be cut` + annotation "(translate to course language)"
- Table headers: `Блок / Слайды / Время` → `Block / Slides / Time` + annotation

**Type 2 — example speech and formatting conventions** (illustrative content):
Replace with English equivalents + note "generate all spoken text in course language":
- `Добрый день. Сегодня мы разбираемся с тем, почему...`
  → `Good morning. Today we are working out why...`
- `[*Указать на заголовок слайда. Пауза 3–5 секунд.*]`
  → `[*Point to the slide title. Pause 3–5 seconds.*]`
- `**Ключевой термин** — это не просто математическое определение, это...`
  → `**Key term** — this is not just a mathematical definition, it's...`
- `⏱ *Контрольная точка: ~8 мин от начала*`
  → `⏱ *Checkpoint: ~8 min from the start*`
- `Итак, сегодня мы разобрали три вещи...` → `So today we covered three things...`
- `[*Не торопиться. Дать студентам записать.*]`
  → `[*Don't rush. Give students time to write.*]`

**Formatting conventions section:**
- `[*Ремарка в скобках курсивом*]` → `[*Stage direction in italic brackets*]` — director's note: where to point, pacing, pause, audience question
- `**Жирный**` → `**Bold**` — term to emphasize verbally
- `⏱ Контрольная точка` → `⏱ Checkpoint` — after each content block, with cumulative time
- `Plain text — то, что говорить` → `Plain text — what to say`

**Tone rules — Russian examples:**
- `"Как вы думаете, что произойдёт если...?"` → `"What do you think will happen if...?"`
- `[*Пауза*]` → `[*Pause*]`
- `[*Дать записать*]` → `[*Let them write*]`
- `"таким образом, можно заключить что"` → `"thus it can be concluded that"`

Add one line at the top of the output format section:
